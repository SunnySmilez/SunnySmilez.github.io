go环境搭建

发表于 2017-02-20 | 分类于 GO

下载go

在列表中选择需要的版本：http://code.google.com/p/go/downloads/list?q=OpSys-Darwin+AND+Type-Installer
将安装包安装到/usr/local目录下：tar -C /usr/local -xzf go.tar.gz

设置环境变量

export PATH=$PATH:/usr/local/go/bin (临时生效)
在/etc/profile文件中添加一行：/usr/local/go/bin （全局有效）
在~/.profile文件中添加一行：/usr/local/go/bin （当前用户生效）

如果需要安装到指定位置

首先安装到指定位置比如是家目录
设置环境变量：export GOROOT=$HOME/go
设置环境变量：export PATH=$PATH:$GOROOT/bin

建立一个文件hello.go，内容如下

package main

import "fmt"

func main() {
    fmt.Printf("hello, world\n")
}

运行

go run hello.go
若看到hello，world信息，那么go已被安装
编译文件：go build test.go
运行文件：./test则也会看到hello，world的信息

技术

技术总结
  • 去年总的技术学习分两个方面，一个是广度的学习也就是进行的linux中级运维知识学习；一个是深度的学习也就是redis源码阅读；广度方面的出发点是想拓宽眼界，对需要用到和可能接触到的东西进行扫盲；深度方面的出发点是觉得很多成熟的开源产品都包含网络，i/o，并发，性能等方面的处理，想要通过熟悉一个较小的框架来对一些基础或者常用的知识进行深入的理解，再者想深入的学习数据存储方面（mysql太大，相对来说redis比较熟悉，成熟还比较小），给自己打造一个特长点
  • 1.在广度学习方面包括linux，nginx一些基本功能，服务，模块配置等，虽然最终形成了一定的博客积累，但是质量偏差，没有太多的自我思考，而且学习之后的效果也十分不理想，最后只留下了大概的印象（没有在工作中用到），仅形成了一定的思路；
  • 总结：在广度学习这一块分为两种情况；1.工作上能用的上的，那就必须花时间认真细学，并且应用到工作中，遇到问题做详细的记录；2.工作上用不上的，给自己增长见识的，提供解决问题的思路，这种只要进行熟悉的，不需要花费太多的时间去了解他的细节，大概知道有它的应用场景和解决什么问题就行了，等到真的用上的时候再反过来进行详细的学习（因为当时学习了依然会忘记）

  • 总结：在一些基础的知识点方面还是很欠缺，找时间还是依然要补回来，比如说c语言，数据结构，算法，操作系统等

思考

关于思考能力15年处于不会思考，16年通过对一部分公众号的阅读（碎片阅读，没有进行整理，这块得改进），以及对身边的人的处事方面进行观察,有所改进，对一些事务能有自己的部分独立看法，但是没有形成自己的一套方法论，这块还需要进行思考总结，目前会按照下面三种方式去思考问题：
1.设身处地思考；这个思考方式举个小例子，比如说当你看到阿里的月饼门事件的报道，你是先看到事件本身然后去抨击阿里的好坏，还是把自己想象成ali的hrbp？或者想象成抢月饼的技术，如果你是他你会如何做？这样的几个角度会很好的锻炼思维，也能多方面的去思考问题；推荐在一些运营的问题上用这样的方式去思考，如果是我，我会怎么做
2.思考事情发生的本质或者说事物产生的原因；举个例子为什么滴滴和快滴进行价格战？为什么最终滴滴和uber要合并？当然有些问题可能按照自己目前的认知水平很难回答，比如说人为什么或者；但是还是要按照这样的思维方式去思考，还是能给独立思考带来很大的帮助
3.对标；这个思路能很好地应用到工作和生活中去，比如说我要做一个催收系统，但是我之前压根不了解他有什么，那么我们就可以去用市面上的产品，总结并结合自己的业务区思考；还有就是对于自己人生目标的设定，我特别崇拜某某，想成为他一样的技术大牛，我就去研究他会什么，然后是如何学习的，通过对他的研究，然后建立自己的一些规划
总结：觉得这些都是一些零碎的思考点，希望能通过一些人或者书籍能够整理出自己对问题的一些解决方法和自己的思考模式，最终能够形成对事物的独立思考

工作

理论
  • 贷款流程：贷前，贷中，贷后
  • 贷前：指的是在给用户进行放款之前的时间
  • 贷中：指的是给用户放款之后和用户还款日期之前的时间
  • 贷后：指的是用户过了还款期限之后的时间
  • 催收系统：属于贷后的行为，主要是针对逾期不还款用户进行电话短信等催收服务，该系统用于协助贷后人员进行逾期订单的催回，以及贷后人员的人效管理
  • 回访系统：包含贷前和贷中部分风险控制策略，提前发现风险点
  • 呼叫中心：主要提供和呼叫相关的服务，具备多个呼叫通道，保障呼叫的可用性（提供服务包含：外呼，群呼，语音呼叫，转接等）
  • CRM：客户关系管理系统，主要用户对客户的管理，以及对订单的实时跟踪
总结
  • 这一年工作中收获的更多的是对于业务的理解能力，做到了知其然知其所以然，也更多的站在了业务方的角度去思考问题，能够从一个更好的角度去思考一个产品，为什么要做（立项的必要性），要做成什么样子（定位），以及对项目周期的控制；

环境变量的设置

发表于 2016-12-21 | 分类于 Linux

永久对所有用户生效

修改/etc/profile
添加如下行(下面的PATH=$PATH…是添加的)，对/etc/profile的不同，添加的位置可能不同
pathmunge () {
    case ":${PATH}:" in
        *:"$1":*)
            ;;
        *)
            if [ "$2" = "after" ] ; then
                PATH=$PATH:$1
            else
                PATH=$1:$PATH
            fi
    esac
        PATH=$PATH:/usr/local/mysql/bin:/usr/local/mysql/sbin:/usr/local/php/sbin:/usr/local/php/bin:/usr/local/nginx/sbin
}
source /etc/profile

永久对当前用户生效（注意要重新登录才生效）

修改~/.bash_profile
PATH=$PATH:$HOME/bin:/usr/local/mysql/bin
source ~/.bash_profile

临时对当前终端生效（终端断开，换将变量就需要重新设置）

终端执行命令
export PATH=$PATH:/usr/local/mysql/sbin

配置client日志收集到本地

*.* /tmp/rsyslog.log

#$IncludeConfig /etc/rsyslog.d/*.conf

<?php
    $message='this is test';
    openlog('this is test, do you know', LOG_ODELAY, LOG_LOCAL7);
    syslog(LOG_DEBUG, $message);
    closelog();
或者在终端运行（local7指定运行级别）
#-p 指定自定义的日志设备，和配置文件的local5.*对应
logger -it error  -p local5.info "hello world"
测试
运行脚本(/usr/local/php/bin/php为你php的命令路径)
/usr/local/php/bin/php /data1/htdocs/rsyslog_input.php
安装php环境，请跳转：安装php运行环境
查看结果
在/tmp/rsyslog.log中查看，会有我们记录的内容

client传输日志到server

在/etc/rsyslog.conf文件中删除一行
*.* /tmp/rsyslog.log
在/etc/rsyslog.conf文件中添加一行
$IncludeConfig /etc/rsyslog.d/*.conf
在/etc/rsyslog.d/添加文件client.conf
需要注意的是target是你的server的地址，配置client传输日志到server文件样例
if ($syslogfacility-text == 'local6') or ($syslogfacility-text == 'local7') or ($syslogtag == 'php:') or ($msg startswith 'PHP') then {
        action(type="omfwd"
                        target="192.168.1.12" port="514" protocol="tcp"
                        queue.type="linkedList"
                        queue.spoolDirectory="/var/spool/rsyslog"
                        queue.fileName="fwd_rsyslog.com"
                        queue.maxDiskSpace="5g"
                        queue.saveOnShutdown="on"
                        action.resumeRetryCount="-1"
              )
                ~
}
按照上述’配置client日志收集到本地’的步骤，配置好server服务器，如果接收到日志说明没有问题，如果有问题，请用tcpdump查看错误原因

配置server日志交给脚本处理

编辑日志处理脚本,/data1/htdocs/rsyslog.php
<?php
while(!feof(STDIN)){
        $line = trim(fgets(STDIN));
        if (empty($line)) {
                continue;
        }

        file_put_contents('/tmp/rsyslog2.log', $line, FILE_APPEND);
}
在/etc/rsyslog.conf文件中删除一行
*.* /tmp/rsyslog.log
在/etc/rsyslog.conf文件中添加一行
$IncludeConfig /etc/rsyslog.d/*.conf
在/etc/rsyslog.d/添加文件server.conf，server收集日志交给脚本处理配置文件样例
module(load="omprog")

template(name="log" type="string" string="%fromhost-ip% %syslogtag% %msg%\n")

*.* action(type="omprog" binary="/usr/local/php/bin/php /data1/htdocs/rsyslog.php" template="log")
*.* ~
重启rsyslog
/etc/init.d/rsyslog restart
在client端写入日志
/usr/local/php/bin/php /data1/htdocs/rsyslog_input.php
在server端查看/tmp/rsyslog2.log的内容，如果有内容写入则整体架构搭建完成，可根据自己的需求去编写日志分析脚本
